üì¶ Installation

Prerequisites

  • FiveM Server
  • MySQL Database
  • Discord Bot (for role checking)
  • oxmysql resource

Step-by-Step Installation

1. Download and Extract

# Place the TD-VehicleDiscordWhitelist folder in your server's resources directory resources/[local]/TD-VehicleDiscordWhitelist/

2. Database Setup

# Import the database.sql file into your MySQL database # This creates all necessary tables and relationships mysql -u username -p database_name < database.sql

3. Configuration

Edit config.lua with your settings:

-- Discord Integration Config.Discord = { BotToken = "YOUR_DISCORD_BOT_TOKEN", GuildId = "YOUR_DISCORD_SERVER_ID", } -- Permissions Config.ManagerPermission = "vehiclewhitelist.manager" Config.AdminPermission = "vehiclewhitelist.admin"

4. Server Configuration

# Add to your server.cfg ensure oxmysql ensure TD-VehicleDiscordWhitelist

5. Permissions Setup

# Add to your server.cfg or permissions file add_ace group.admin vehiclewhitelist.manager allow add_ace group.admin vehiclewhitelist.admin allow
Installation Complete! Restart your server and test the system with /vehiclewhitelist

Basic Usage

1. Opening the Interface

  • Use /vehiclewhitelist or /vw command
  • Press F3 keybind (configurable)
  • Navigate between tabs: Vehicles, Owned Vehicles, Management

2. Vehicle Garage

  • View all available vehicles
  • Search for specific vehicles
  • Preview vehicles before spawning
  • Spawn vehicles you have access to

3. Vehicle Packs

  • Browse vehicles organized by packs
  • Click on packs to view their vehicles
  • See pack descriptions and requirements

4. Management (Manager/Admin Only)

  • Create and edit vehicle packs
  • Add individual vehicles to whitelist
  • Set Discord role requirements
  • Manage thumbnails and descriptions
  • Hide/show vehicles and packs

Creating Vehicle Packs

  1. Go to the Management tab
  2. Click "Create Pack"
  3. Fill in the pack details:
    • Title: Name of the pack
    • Description: What vehicles are in this pack
    • Thumbnail: Image URL for the pack
    • Discord Role: Role required to access pack vehicles

Adding Vehicles

  1. Go to Management > Manage Vehicles
  2. Click "Add Vehicle"
  3. Fill in vehicle details:
    • Vehicle Model: GTA V model name (e.g., "adder")
    • Vehicle Hash: Auto-generated or manual
    • Vehicle Name: Display name
    • Description: Vehicle description
    • Pack: Assign to a pack (optional)
    • Discord Role: Role required (optional)

Key Relationships

  • Both packs and vehicles can have Discord role requirements
  • Spawn tracking logs all vehicle spawns for monitoring
  • Discord roles are cached to prevent API rate limits

üîå Events & Callbacks

This section documents the FiveM events and NUI callbacks used by the TD-VehicleDiscordWhitelist resource for integration with other scripts.

Client Events

Opening Whitelist

-- Open whitelist with permissions (server to client) TriggerClientEvent('td-vehiclewhitelist:openWhitelistWithPermissions', source, permissions, packs, vehicles, playerRoles)

Vehicle Spawning

-- Spawn vehicle (server to client) TriggerClientEvent('td-vehiclewhitelist:spawnVehicle', source, vehicleModel)

Preview Mode

-- Enter preview mode (server to client) TriggerClientEvent('td-vehiclewhitelist:enterPreviewMode', source, data) -- Exit preview mode (server to client) TriggerClientEvent('td-vehiclewhitelist:exitPreviewMode', source)

Server Events

Player Events

-- Register these events in your scripts (client to server) RegisterNetEvent('td-vehiclewhitelist:openWhitelist') RegisterNetEvent('td-vehiclewhitelist:spawnVehicle') RegisterNetEvent('td-vehiclewhitelist:previewVehicle') RegisterNetEvent('td-vehiclewhitelist:exitPreviewMode')

Management Events

-- Manager/Admin functions (client to server) RegisterNetEvent('td-vehiclewhitelist:createPack') RegisterNetEvent('td-vehiclewhitelist:updatePack') RegisterNetEvent('td-vehiclewhitelist:deletePack') RegisterNetEvent('td-vehiclewhitelist:addVehicle') RegisterNetEvent('td-vehiclewhitelist:updateVehicle') RegisterNetEvent('td-vehiclewhitelist:removeVehicle') RegisterNetEvent('td-vehiclewhitelist:refreshDiscordRoles')

NUI Callbacks

UI to Client Communication

These callbacks handle communication between the HTML/JavaScript UI and the FiveM client script:

Callback Purpose Parameters
spawnVehicle Spawn a vehicle from the UI {vehicleId}
previewVehicle Preview a vehicle in 3D {vehicleId}
createPack Create a new vehicle pack {title, description, thumbnail, discordRoleId}
addVehicle Add vehicle to whitelist {vehicleModel, vehicleHash, vehicleName, etc.}
getDiscordRoles Get available Discord roles None
refreshRoles Refresh Discord roles None
Note: This is a FiveM resource script, not a web API. These events and callbacks are for FiveM server/client communication and NUI integration.

üîß Troubleshooting

Common Issues

Discord Role Not Working

  • Verify bot token and server ID are correct in config.lua
  • Check bot permissions in Discord server
  • Ensure role IDs are correct (use Discord Developer Mode)
  • Try refreshing roles with /refreshroles

Vehicles Not Spawning

  • Check if vehicle model exists in GTA V
  • Verify player has required Discord role
  • Check spawn location for obstructions
  • Ensure cooldown period has passed (30 seconds)
  • Check if player is in a vehicle (passenger seats blocked)

UI Not Opening

  • Verify permissions are set correctly in server.cfg
  • Check console for JavaScript errors
  • Ensure resource is started properly
  • Check if oxmysql dependency is loaded

Debug Mode

Enable Debug Mode

-- In config.lua Config.Debug = true

This will provide detailed console output for troubleshooting including:

  • Discord role checking process
  • Vehicle spawn attempts
  • Database queries
  • Permission checks
  • UI state changes

Console Commands

/tdvehiclewhitelist_debug
Debug command (when debug mode enabled)

Performance Tips

  • Discord roles are cached for 5 minutes to prevent API rate limits
  • Use database indexes for optimal query performance
  • Enable debug mode only when troubleshooting
  • Monitor spawn tracking table size and clean up old entries

Support

Getting Help:
  • Check the console for error messages
  • Verify your configuration settings
  • Ensure all dependencies are properly installed
  • Test with debug mode enabled for detailed logging
