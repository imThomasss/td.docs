TD-ApplicationPortal Documentation

Complete application management system for Discord communities, featuring OAuth2 authentication, application review workflows, and administrative tools.

Windows VPS Node.js Discord OAuth2 SSL Ready

Overview

TD-ApplicationPortal is a comprehensive web-based application management system designed for Discord communities. It provides a secure, professional platform for managing member applications with Discord OAuth2 integration, role-based permissions, and an intuitive review workflow.

Discord Integration

Seamless Discord OAuth2 authentication with role synchronization and permission management.

Application Management

Complete application lifecycle management with review workflows and status tracking.

Secure & Licensed

Built-in license verification system and SSL certificate support for secure connections.

Role-Based Access

Granular permission system with admin, reviewer, and applicant roles.

Requirements

Server Requirements

Before you begin, ensure you have the following:

  • Windows VPS - Windows Server or equivalent hosting solution
  • Domain/Subdomain - Pointing to your VPS IP (e.g., apply.thomasdevelopments.com)
  • Node.js - Latest LTS version recommended
  • MariaDB - Database server for storing application data
  • HeidiSQL - More user friendly database management tool

Required Software

Install these applications on your Windows VPS:

  • Node.js - JavaScript runtime for the application
  • Certify The Web - SSL certificate management tool
  • MariaDB - Recommended database (or alternative SQL database)
  • HeidiSQL - Database management tool

Discord Requirements

  • Discord Server - For OAuth2 integration
  • Discord Bot Application - Created via Discord Developer Portal
  • Administrator Permissions - For bot configuration

Installation

Step 1: Database Setup

Set up your database using HeidiSQL:

  1. Open HeidiSQL and connect to your database server
  2. Create a new database for the application portal
  3. Locate the database.sql file in the project folder
  4. Load and run the SQL file to create all required tables

Database Tables

The SQL file will create tables for:

  • User accounts and authentication
  • Application submissions
  • Application reviews and comments
  • Role and permission management

Step 2: Firewall Configuration

Open the required ports on your Windows VPS firewall:

Required Ports:
• Port 80 (HTTP)
• Port 443 (HTTPS)
• Port 6000 (Application Backend)
• Port 6001 (Application Frontend)

Windows Firewall Setup:

  1. Open Windows Defender Firewall → Advanced Settings
  2. Click Inbound Rules → New Rule
  3. Select Port → Next
  4. Add each port (80, 443, 6000, 6001)
  5. Allow the connection → Finish

Hosting Provider Firewall

If using a hosting provider, you may also need to open these ports through their control panel or firewall management interface.

Step 3: Install Dependencies

Navigate to the project folder and install Node.js dependencies:

cd D:\TD-Scripts\TD-ApplicationPortal
npm install

This will install all required packages for both frontend and backend.

Configuration

Environment File Setup

Configure your application by editing the .env file:

  1. Locate the .env file in the project root folder
  2. Open it with a text editor (Notepad, VS Code, etc.)
  3. Update the following settings according to your environment:
# Database Configuration
DB_HOST=localhost
DB_USER=your_database_user
DB_PASSWORD=your_database_password
DB_NAME=application_portal

# Server Configuration
PORT=6000
FRONTEND_PORT=6001
DOMAIN=apply.yourdomain.com

# Discord OAuth2
DISCORD_CLIENT_ID=your_client_id
DISCORD_CLIENT_SECRET=your_client_secret
DISCORD_BOT_TOKEN=your_bot_token
DISCORD_GUILD_ID=your_guild_id
DISCORD_REDIRECT_URI=https://apply.yourdomain.com/auth/discord/callback

# License System
LICENSE_KEY=your_license_key
LICENSE_WEBHOOK=your_webhook_url

License Key Setup

Generate and configure your license key:

  1. Join our Discord server: https://discord.gg/gy6V3kaJr4
  2. Navigate to the Generate License Key channel
  3. Click the button to create your license key
  4. Enter your domain (e.g., apply.thomasdevelopments.com)
  5. Enter your VPS IP address (must match DNS record)
  6. Copy the generated license key and webhook URL
  7. Paste both values into your .env file

Important Notes

  • Your license is tied to your domain and IP address
  • If either changes, contact our staff on Discord with proof of ownership
  • Keep your license key private and secure

Discord Bot Configuration

Set up Discord OAuth2 and bot integration:

1. Create Discord Application

  1. Go to Discord Developer Portal
  2. Click "New Application" and give it a name
  3. Create a bot under the "Bot" tab

2. Configure OAuth2

  1. Go to OAuth2 → Redirects
  2. Add your redirect URL: https://yourdomain.com/auth/discord/callback
  3. This URL must match exactly what's in your .env file
  4. Go to OAuth2 → General
  5. Copy your Client ID and Client Secret
  6. Paste them into your .env file

3. Bot Configuration

  1. Go to the Bot tab
  2. Enable all Privileged Gateway Intents:
    • Presence Intent
    • Server Members Intent
    • Message Content Intent
  3. Copy the Bot Token
  4. Paste it into your .env file as DISCORD_BOT_TOKEN

4. Get Guild ID

  1. Enable Developer Mode in Discord (User Settings → Advanced → Developer Mode)
  2. Right-click your server → Copy Server ID
  3. Paste it into your .env file as DISCORD_GUILD_ID

5. Invite Bot to Server

  1. Go to OAuth2 → URL Generator
  2. Select scopes: bot
  3. Select permissions: Administrator
  4. Copy the generated URL
  5. Open it in your browser and invite the bot to your server

SSL Certificate Setup

Configure SSL certificates using Certify The Web:

1. Request Certificate

  1. Open Certify The Web on your VPS
  2. Click "Create New Certificate"
  3. Add your domain (e.g., apply.thomasdevelopments.com)
  4. Click "Request Certificate"
  5. Wait for certificate validation and issuance

2. Export Certificate Files

Configure automated certificate export:

Export Certificate Chain:

  1. Go to the Tasks tab → Add
  2. Select "Export Certificate"
  3. Task Name: Certificate
  4. Destination: D:\TD-Scripts\TD-ApplicationPortal\ssl\certificate.crt
  5. Export As: PEM – Full Certificate Chain (excluding key)
  6. Click OK

Export Private Key:

  1. Add another task → Select "Export Certificate"
  2. Task Name: Key
  3. Destination: D:\TD-Scripts\TD-ApplicationPortal\ssl\private.key
  4. Export As: PEM – Private Key (.key)
  5. Click OK

3. Run Export Tasks

  1. Run both export tasks to generate the certificate files
  2. Verify both files exist in your ssl folder:
    • certificate.crt
    • private.key

Certificate Renewal

Important: SSL certificates must be renewed every 90 days. Certify The Web will handle automatic renewal, but ensure your export tasks run after each renewal to update the files.

Running the Application

Starting the Application

Launch the application using npm scripts:

  1. Open Command Prompt or Terminal
  2. Navigate to the project folder:
    cd D:\TD-Scripts\TD-ApplicationPortal
  3. Start the application:
    npm start

This command will:

  • Build the frontend React application
  • Start the backend Node.js server
  • Serve the application on configured ports
  • Verify your license key

Managing the Application

Stopping the Application

Press Ctrl + C in the terminal where the application is running.

Restarting the Application

Simply run npm start again after stopping.

Development Mode

For development with hot-reload:

npm run dev

Accessing Your Application

Once started, you can access the application:

👉 https://yourdomain.com

You should see:

  • License verification confirmation
  • Discord login button
  • Application portal interface

First Time Setup

The first user to log in will automatically receive administrator privileges. Subsequent users will need to be granted permissions through the admin panel.

Database Structure

Main Tables

The application uses the following database tables:

users

Stores user accounts and Discord information:

  • Discord ID and authentication tokens
  • Username, discriminator, and avatar
  • Role assignments (admin, reviewer, user)
  • Account creation and last login timestamps

applications

Stores application submissions:

  • Application type and form responses
  • Current status (pending, approved, denied)
  • Submission timestamp and last update
  • Associated user ID

application_reviews

Tracks review history:

  • Reviewer user ID
  • Review decision and notes
  • Review timestamp
  • Associated application ID

application_comments

Internal comments on applications:

  • Comment author (reviewer)
  • Comment text and timestamp
  • Associated application ID
  • Visibility settings

Troubleshooting

Common Issues

License Verification Failed

Problem: Application won't start due to license verification failure

Solutions:

  • Verify your license key is correctly entered in the .env file
  • Ensure your domain and IP address match what was registered
  • Check that your server has internet connectivity
  • Contact support on Discord if your IP or domain changed

Discord OAuth2 Not Working

Problem: Can't log in with Discord

Solutions:

  • Verify Discord Client ID, Secret, and Bot Token are correct
  • Check that redirect URI in Discord matches your .env exactly
  • Ensure bot is invited to your server with correct permissions
  • Verify all Privileged Gateway Intents are enabled
  • Check Guild ID is correct

Database Connection Errors

Problem: Can't connect to database

Solutions:

  • Verify database credentials in .env file
  • Ensure MariaDB/MySQL service is running
  • Check database exists and tables were created
  • Verify database user has proper permissions

SSL Certificate Issues

Problem: HTTPS not working or certificate errors

Solutions:

  • Verify certificate files exist in the ssl folder
  • Check file names: certificate.crt and private.key
  • Ensure certificate export tasks ran successfully
  • Renew certificate if expired (90-day validity)
  • Verify domain matches the certificate domain

Ports Already in Use

Problem: Application won't start due to port conflicts

Solutions:

  • Check if another application is using ports 6000 or 6001
  • Stop any conflicting services
  • Change port numbers in .env file if needed
  • Restart your server if necessary

Firewall Blocking Connections

Problem: Can't access the application from outside

Solutions:

  • Verify all required ports are open (80, 443, 6000, 6001)
  • Check both Windows Firewall and hosting provider firewall
  • Test with firewall temporarily disabled to isolate the issue
  • Ensure inbound rules allow connections

Getting Logs

View application logs for debugging:

  • Backend logs appear in the terminal where you ran npm start
  • Check browser console for frontend errors (F12 → Console)
  • Database errors will appear in MariaDB logs
